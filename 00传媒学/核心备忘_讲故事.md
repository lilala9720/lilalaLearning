# 核心备忘：讲故事

> 每次写脚本前看一眼。

## 你的旧模式 vs 新模式

**旧模式：** 概念 → 解释 → 配例子 → 给结论
**观众体验：** "嗯有道理" → 划走 → 三秒后忘了

**新模式：** 具体场景/冲突 → 在冲突中挣扎 → 在挣扎中领悟 → 概念自然浮出来
**观众体验：** "这不就是我吗？" → "然后呢？" → "原来是这样！" → 转发

## 核心区别：谁在前

| | 旧做法 | 新做法 |
|---|---|---|
| 开头 | 概念在前："NPD的核心特征是……" | 场景在前："我哭着问我爸……" |
| 中间 | 例子服务概念 | 概念服务故事 |
| 结尾 | 总结道理 | 角色变化了（道理藏在变化里） |

## 你为什么之前没做到？四个原因

1. **短视频训练了你"点到即止"** — 你习惯压缩一切，把故事也压缩成了结论
2. **你是概念先行的思维方式** — 你脑子里先浮现"原理"不是"画面"，但观众的脑子是反过来的
3. **你在回避挣扎的部分** — 挣扎是痛苦的，你本能地跳过。但挣扎恰恰是故事的心脏，是观众产生共鸣的地方
4. **你觉得观众要的是"答案"不是"过程"** — 观众点进来确实是为了答案，但让她们留下来并分享的，是过程中的共鸣

## 每次写脚本前问自己的一个问题

> "关于这个主题，我自己（或身边的人）经历过的最痛的一个瞬间是什么？"

从那个瞬间开始写。概念后面再说。

先有痛，再有理论。观众跟你走的是同一条路。
