# 产品经理思维 - 第 3 课：需求分析——痛点、痒点、爽点

## 学习目标

学完这一课，你能：
- 区分三种需求类型，判断一条内容到底在满足哪种需求
- 针对每种需求类型，使用对应的选题策略和内容结构
- 规划内容节奏：什么时候打痛点、什么时候种痒点、什么时候给爽点

---

## 内容讲解

### 一、先把上节课的发现拎出来

你已经有了三种武器的雏形：

- **痛点内容**（NPD 为什么要打压你 → 2000+ 赞）
- **痒点内容**（画一个她能想象到的小画面）
- **爽点内容**（给她一个今天就能用的方法）

但你之前是"撞上"的。这节课的目标是：**让你能主动选择用哪种武器，而不是靠运气。**

### 二、三种需求，三种情绪引擎

梁宁在《产品思维30讲》里提出一个框架：人的需求可以分为痛点、痒点、爽点。我们把它翻译成你的战场。

#### 痛点：恐惧驱动

> **"我怕……"**

痛点的底层情绪是**恐惧**——怕失去、怕被抛弃、怕自己是错的、怕这辈子就这样了。

你用户的痛点你已经挖到了：

- 怕是自己的错（核心痛点）
- 怕离婚之后更惨
- 怕恨父母说明自己不孝
- 怕停下来全家就垮了

**痛点内容的作用：一把抓住她的注意力。** 人对恐惧的反应是最快的——这就是为什么你的 NPD 视频断层领先。它直接戳中了"我是不是被人操控了而自己不知道"的恐惧。

**痛点内容的结构：**

```
戳痛（前3秒）→ 揭示真相（主体）→ 落回她身上（结尾）
```

- **前3秒**：用"对立视角"或"揭示动机"开头，制造"咯噔"
  - 例："你有没有发现，每次吵完架，道歉的人永远是你？"
- **主体**：拆解这个现象背后的心理学机制
  - 例：为什么有些人从不道歉——因为道歉意味着承认自己不完美，NPD 无法承受这个
- **结尾**：一句话回到她身上
  - 例："所以不是你做错了什么，是你一直在替他承担他该承担的情绪。"

#### 痒点：向往驱动

> **"我想……"**

痒点的底层情绪是**向往**——想成为某种人、想过某种生活、想拥有某种能力。

痛点是"离开现在的苦"，痒点是"靠近想要的样子"。

你用户的痒点是什么？不是"财务自由"那种大饼。是这种很小、很具体的向往：

- 想要一个晚上不用任何人交代的周末
- 想要说"不"之后对方居然尊重了她
- 想要一段不用猜对方心思的关系
- 想要照镜子的时候觉得自己还不错

**痒点内容的作用：让她留下来。** 痛点让她点进来，痒点让她关注你——因为她觉得"这个人知道我想要什么"。

**痒点内容的结构：**

```
画面（前3秒）→ 反差（为什么你现在得不到）→ 一步（最小的一个动作）
```

- **前3秒**：描述那个她向往的具体画面
  - 例："你有没有想象过，有一天你说'我不想去'，然后——就真的没去？"
- **反差**：为什么她现在做不到——不是她不行，是她的模式在挡她
  - 例：从小被训练成"乖巧 = 安全"，所以拒绝别人会触发她的危险警报
- **一步**：给她一个很小的、今天就能做的第一步
  - 例："今天试一次：有人让你做你不想做的事，别说'好的'，说'我想想'。只这三个字就够了。"

#### 爽点：即时满足

> **"我终于……"**

爽点的底层情绪是**即时满足**——当场就有用、马上就见效、做完就舒服。

你用户的爽点场景：

- 学了一句话，今天就用上了，对方真的闭嘴了
- 看到一个分析，突然想通了困扰自己很久的事
- 听完你一期视频，憋了很久的情绪终于哭出来了

**爽点内容的作用：让她信你。** 痛点让她点进来，痒点让她关注，爽点让她觉得"这个人说的东西真的有用"——这是转付费的关键一步。

**爽点内容的结构：**

```
场景（前3秒）→ 工具/话术（主体）→ 效果预告（结尾）
```

- **前3秒**：一个她每天都遇到的具体场景
  - 例："老公又说'你怎么什么都做不好'——你通常怎么回？"
- **主体**：给一个她能直接用的工具或话术
  - 例："试试这句：'你可以对我有意见，但不可以用这种方式说。' 不解释、不争论、不道歉。就这一句。"
- **结尾**：告诉她用完之后会发生什么
  - 例："他大概率会愣住。因为他习惯了你接招。你第一次不接，游戏规则就变了。"

### 三、三种需求怎么配比

不是三种选一种，是**三种按节奏配合**。

```
痛点（引流）→ 痒点（留人）→ 爽点（转化）→ 痛点（再引流）……
```

如果你一周发 3 条内容：

| 第1条 | 第2条 | 第3条 |
|-------|-------|-------|
| 痛点：揭示动机、戳中恐惧 | 痒点：画小画面、种下向往 | 爽点：给工具、给话术 |
| 拉新用户进来 | 让她关注你 | 让她信任你 |

如果你一周只发 1 条，就在一条视频里把三层都做了：

```
痛点开头（前3秒戳痛）→ 痒点中段（描述"之后"的画面）→ 爽点结尾（给一个动作）
```

你的边界感那条视频其实就是这个结构：
- 痛点：我爸踹门的故事（恐惧）
- 痒点：发现"真正的问题是什么"的那个瞬间（向往）
- 爽点：把问题从"我是不是不够好"换成"这到底是不是我的责任"（工具）

——所以它效果好。不是因为一个点打中了，是三层都打中了。

### 四、一个常见的坑：痛点做多了会怎样

如果你每条都是"他为什么打压你""他为什么控制你""他为什么冷暴力"——

前几条会爆。然后：

1. 评论区变成集体控诉大会
2. 吸引来一批只想发泄不想改变的人
3. 她看多了会麻——"又来了，换个角度的骂他而已"
4. 算法给你打标签："情绪宣泄号"，推流质量下降

**痛点是火，但火烧多了会把房子烧掉。** 你需要痒点和爽点来平衡——让她知道你不只是理解她的痛，你还能带她往前走。

这也是你从"内容号"走向"付费产品"的路径：

> 痛点让她知道你 → 痒点让她想跟你走 → 爽点让她相信你能带路 → 付费让她正式上路

### 五、用你的真实内容验证

回头看你的 25 条视频，按这三类重新归类：

- 哪些是痛点内容？数据怎样？
- 哪些是痒点内容？数据怎样？
- 哪些是爽点内容？数据怎样？
- 有没有一条视频里同时包含了两层甚至三层？那条数据是不是比只有一层的好？

这个分类做完，你就知道自己的内容库里缺什么了。

---

## 小结

1. **痛点 = 恐惧**：戳中她的怕，让她停下来 → 拉新
2. **痒点 = 向往**：画她想要的画面，让她跟着你 → 留存
3. **爽点 = 即时满足**：给她能用的东西，让她信你 → 转化
4. **三种交替用**：痛点引流 → 痒点种草 → 爽点转化，形成闭环
5. **痛点不能滥用**：全做痛点会变成情绪宣泄号，失去商业价值
6. **最好的内容是三层合一**：一条视频里痛点开头、痒点中段、爽点结尾
